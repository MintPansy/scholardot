# 백엔드 수정 없이 프론트엔드만으로 진행 가능한 개선 목록

논문/디벨롭 시 **백엔드는 최대한 건드리지 않고** FE(`fe-paper-reader`)만 수정해서 할 수 있는 항목들입니다.

---

## 1. API URL·환경 설정 통일 (즉시 적용 가능)

- **현재**: `app/services/document.ts`의 `getTranslationStatus` 안에 `https://be-paper-dot.store` 하드코딩됨.
- **개선**: `getApiUrl()` 사용으로 통일.
- **작업**: 해당 한 줄만 수정하면 됨. BE 변경 없음.

---

## 2. 번역 결과 검색 UX 강화

- **현재**: 읽기 화면에 검색창 + 하이라이트는 이미 구현됨 (`ReadList.tsx`의 `searchQuery`, `highlightMatches`).
- **FE만으로 가능한 보강**:
  - 검색 결과 **개수 표시** (예: "총 12건")
  - **첫 매치 / 다음 / 이전** 버튼으로 해당 문단으로 **스크롤 이동**
  - (선택) 검색어가 비어 있을 때 placeholder/안내 문구 정리
- **데이터**: 이미 `getTranslatedDocument`로 받은 번역 쌍을 클라이언트에서 검색하므로 BE 수정 불필요.

---

## 3. 테스트·CI (FE만)

- **단위/통합 테스트**: 컴포넌트·유틸·훅 테스트 (Jest + React Testing Library 등). API는 모킹.
- **E2E(선택)**: Playwright 등으로 업로드 → 읽기 화면 진입 등 시나리오. 기존 백엔드 URL 그대로 사용.
- **CI**: GitHub Actions에서 FE만 `npm run build`, `npm test` 실행. BE 코드/빌드 건드리지 않음.

---

## 4. UI/UX·접근성 (FE만)

- **다크 모드**: CSS 변수/테마 + 토글. 상태는 localStorage 등으로 유지.
- **폰트 크기 조절**: 읽기 화면에서만 크기 선택 (예: 작게/보통/크게). CSS/상태만.
- **접근성**: 키보드 네비게이션(탭 순서, Enter/Space), `aria-label` 보강, 포커스 링 등. 전부 마크업/이벤트만.
- **로딩/에러 UI**: 스켈레톤, 재시도 버튼, 메시지 정리. BE API 시그니처는 그대로.

---

## 5. 메모·하이라이트 “로컬 전용” 버전

- **목표**: 문단 단위 메모/하이라이트를 **저장·복원**하고 싶지만, 당장은 BE를 건드리지 않음.
- **방법**: `localStorage`(또는 `sessionStorage`)에 `documentId` 기준으로 메모/하이라이트 JSON 저장. 읽기 화면 진입 시 로드해서 표시.
- **한계**: 기기/브라우저가 바뀌면 사라짐. 나중에 BE에 메모/하이라이트 API가 생기면 같은 UI로 연동만 추가하면 됨.

---

## 6. 번역문 내보내기 (클라이언트만)

- **기능**: 이미 받은 번역 쌍을 **TXT 또는 JSON**으로 다운로드.
- **구현**: 읽기 화면에 "내보내기" 버튼 → `getTranslatedDocument`로 이미 갖고 있는 데이터(또는 state)를 Blob으로 만들어 `download` 링크. BE 신규 API 불필요.

---

## 7. 반응형·모바일

- **읽기 화면**: 사이드바 접기/펼치기, 작은 화면에서 목차/썸네일 레이아웃 조정.
- **전반**: Tailwind/CSS만으로 breakpoint 조정. BE 변경 없음.

---

## 8. (참고) 백엔드 수정이 필요한 항목

| 항목 | 이유 |
|------|------|
| 실시간 번역 진행률(SSE) | BE에 SSE 엔드포인트(`/documents/{id}/translation-events` 등) 필요. 현재는 폴링만 사용 가능. |
| 메모·하이라이트 서버 저장 | BE에 메모/하이라이트용 테이블·CRUD API 필요. 위 5번은 그 전 단계로 로컬만 저장. |

---

## 추천 진행 순서 (FE만)

1. **API URL 통일** (약 5분)
2. **검색 UX 강화** (스크롤 이동, 결과 개수) — 논문 “구현” 소재로도 좋음
3. **다크 모드 또는 폰트 크기** — 사용성·논문 실험 시나리오로 활용 가능
4. **번역문 내보내기** — 부가 기능으로 설명하기 쉬움
5. **FE 테스트 + CI** — 품질·회귀 방지

이 순서는 서로 의존성이 없으므로, 원하는 것부터 골라서 적용하면 됩니다.
